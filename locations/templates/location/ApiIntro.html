Flirtjar API      Documentation

Introduction

This is an intro to how FlirtJar Api documentation is formated and how to use it.

Api documentation is made with the help of an open tool named Swagger which is an OpenApi specification for RestAPi documentation. This web page is not the actual documentation but rather an intoduction to the critical part of the api such as its Authentication, Response, Errors etc.

FlirtJar Api is basically divided into three parts Users, Location and Profile, Each part groups all the related api endpoint. User section contains all the endpoints related to users as well as authentication, Location section contains endpoints for location based services and Profile section contains endpoints related user profile such as likes or dislike, views, gifts, etc...

API Response

Response returned by the Api is divided into two parts Errors and Result. whether response is successfull or unscuccessful, Both Errors and Result field will always be present in the response of returned by Api.

1. Basic Response Example :

							
{
	"errors": {},
	"result": {}
}
(errors and result are always present in Response.)

2. If Response Is Not Successful:

If response is unsucessfull (may be bacause of wrong data of json error) then all the errors can be found in 'errors' field and result field will be empty in this case.
Example :
							
{
	"errors": [
		"email: enter a valid email address."
	],
	"result": {}
}

2. If Response Is Successful:

If response is sucessfull then errors field will be empty and all the request result can be found in result field.
Example :
							
{
	"errors": {},
	"result": {
		"id": 2294,
		"resp": "success"
	}
}


Authentication

Api uses Token Authentication and client must get a Token before accessing the Api. Client can get a Token by creating an account with the api which can be used for making further api calls.

Token must be passed in 'Authorization' Header of every request else '401 Unauthorized' error will be returned.
Token should be passed as : 'Authorization: Token <your_token>'

FlirtJar Api support only social authentication which must be done from client side and user information fetched from social login should be passed at the time of account creation.

Api doesn't have any login system if user want to re-login(may be after re-installing the app) then client must send his or her credential atleast email and oauth_id to the same url of creating account and if credential mathch then user profile information will be returned.

Parameter Format

Three types of parameter is accepted be the Api :

  • Post Body : Json data that is passed with post body of the request.
  • Query Parameter : Queries that are passed at end of url that start ? in urls.
  • URL Parameter : Parameter which is passed within the url such as /api/uses/{id}
